# Экран просмотра проекта

## Когда показывается
Открывается при выборе проекта из списка на экране **«Проекты»**.

## Назначение
Показать подробную информацию о выбранном проекте: основные данные, статус, описание, требования и команду. Также дать быстрые действия (например, перейти в GitHub и открыть статистику).

## Структура экрана

### 1) Шапка
- Кнопка **назад** (стрелка).
- Заголовок: **«Проекты»**.

### 2) Карточка проекта
Содержит ключевые данные:
- Дата (слева).
- Название проекта.
- Иконка обновления/перезагрузки данных.
- Плашки/поля с датами:
  - **Срок записи на проект**
  - **Срок реализации проекта**
- Информация о заказчике.

### 3) Дополнительная информация
- Контакты/заказчик (в виде небольших плашек).
- Теги/стек проекта (в виде чипсов).
- Статус: **«Назначена команда»**.

### 4) Описание проекта
- Заголовок: **«Описание проекта»**
- Текстовое описание проекта.

### 5) Требования проекта
- Заголовок: **«Требования проекта»**
- Нумерованный список требований.

### 6) Требования для исполнителей
- Заголовок: **«Требования для исполнителей»**
- Чипсы с навыками/требованиями.

### 7) Команда
- Карточка **«Команда»** со списком участников и их ролями.

## Выпадающее меню действий
Справа внизу — группа кнопок:
- **GitHub**
- **Статистика проекта**

## Ограничение для неавторизованного пользователя
Если пользователь **не авторизован**, он видит информацию **только до блока «Требования проекта» (не включительно)** — то есть карточку проекта, статус и **«Описание проекта»**. Всё, что начинается с **«Требования проекта»** и ниже, скрыто.
