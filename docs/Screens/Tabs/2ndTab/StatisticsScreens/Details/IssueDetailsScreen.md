# Экран «Issue» — подробно

## Когда показывается
Открывается из экрана статистики по кнопке **«Подробнее»** в блоке «Issue».

## Назначение
Показать сводные показатели по issues за выбранный период, сравнение участников, список конкретных issues, а также вклад создателей/исполнителей и популярные метки.

## Структура экрана

### 1) Шапка
- Кнопка **назад** (стрелка).
- Заголовок: **«Issue»**.

### 2) Параметры отчёта
- **Выбор репозитория** (выпадающий список).
- **Выбор периода** (интервал дат “с…по…”).
- **Выбор участника** (выпадающий список) — отображается в контексте проекта/команды.

### 3) Сводные показатели
Карточки с ключевыми значениями:
- **Всего Issue**
- **Ср. время жизни Issue**
- **Открытых Issue**
- **Закрытых Issue**
- Подсказка “сколько осталось закрыть” (например, **«Закройте ещё 4 Issue»**).
- **Место в рейтинге**

### 4) Сравнение участников
- Блок **«Количество назначенных Issue (открытые/закрытые)»** — список участников с их значениями (у текущего пользователя пометка **«(Вы)»**).

### 5) Список Issues
- Заголовок блока: **«Список Issues»**
- Действие **«Смотреть все»** — открывает полный список.
- Карточки issues с данными:
  - название issue
  - метка/тег (например, **deploy**)
  - статус (например, **closed**)
  - номер issue
  - создатель
  - назначенные исполнители
  - дата создания и дата закрытия
  - мини-статистика (иконки комментариев/событий и т.п.)
  - ссылка на issue

### 6) Создатели и исполнители
- Блок **«Создатели Issues»** — список пользователей и сколько issues они создали.
- Блок **«Исполнители Issues»** — список пользователей и сколько issues они закрыли.

### 7) Метки
- Блок **«Метки»** — чипсы популярных лейблов (например: `deploy`, `bug`, `tests`) с количеством issues по каждому.

### 8) Итоги
- **Место в команде**
- Итоговая **оценка Issue**
