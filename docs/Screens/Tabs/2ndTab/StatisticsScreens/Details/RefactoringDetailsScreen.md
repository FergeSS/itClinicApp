# Экран «Рефакторинг» — изменчивость кода (подробно)

## Когда показывается
Открывается из экрана статистики по кнопке **«Подробнее»** в блоке «Изменчивость кода».

## Назначение
Показать, какие файлы изменялись чаще всего, как распределены изменения, и как пользователь/участники выглядят в сравнении по этому показателю.

## Структура экрана

### 1) Шапка
- Кнопка **назад** (стрелка).
- Заголовок: **«Рефакторинг»**.

### 2) Параметры отчёта
- **Выбор репозитория** (выпадающий список).
- **Выбор периода** (интервал дат “с…по…”).
- **Выбор участника** (выпадающий список) — отображается, когда экран открыт в контексте проекта/команды.

### 3) Статистика по файлам
- Блок **«Статистика по файлам»** с таблицей:
  - колонки: **Файл** / **Кол-во**
- Действие **«Смотреть все»** — открывает полный список файлов.

### 4) Сводные показатели
- **Изменено файлов** (количество затронутых файлов).
- **Место в рейтинге**.
- Список **«Количество измененных файлов»** по участникам (с пометкой **«(Вы)»** у текущего пользователя).

### 5) Распределение изменений
- Кольцевая диаграмма **«Распределение изменений файлов»** — показывает, сколько файлов попадает в группы по числу изменений (например: 1 изменение, 2–3, 4–5, 6–7, 8–10 и т.д.).
- Легенда с количеством файлов в каждой группе.

### 6) Итоги
- Самый часто изменяемый файл (выделен названием, например **`SRC/CLIENT.TS`**).
- **Место в команде**.
- Итоговая **оценка изменчивости кода**.
