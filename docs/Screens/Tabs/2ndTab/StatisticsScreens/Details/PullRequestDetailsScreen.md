# Экран «Pull Request» — подробно

## Когда показывается
Открывается из экрана статистики по кнопке **«Подробнее»** в блоке «Pull Requests».
В будущих версиях перед формированием отчёта будет доступен шаг настройки расчёта именно для этого блока (как считать показатели по Pull Requests).

## Назначение
Показать динамику PR, сводные показатели по времени жизни PR, сравнение участников, распределение по длительности и список конкретных PR (с быстрыми и медленными).

## Планы на будущие версии
- **Настройка расчёта блока** — отдельный шаг перед экраном детализации, где можно будет задать правила/параметры расчёта для блока «Pull Requests» (например, какие статусы учитывать, как считать время жизни, фильтры, исключения и т. п.).

## Структура экрана

### 1) Шапка
- Кнопка **Назад** (стрелка).
- Заголовок: **«Pull Request»**.

### 2) Параметры отчёта
- **Выбор репозитория** (выпадающий список).
- **Выбор периода** (интервал дат “с…по…”).
- **Выбор участника** (выпадающий список) — отображается в контексте проекта/команды.

### 3) График
- Блок **«График Pull Requests»** с динамикой по датам.

### 4) Сводные показатели
- **Дни** — среднее время жизни Pull Request.
- **Всего Pull Request**.
- **Место в рейтинге**.
- Список **«Количество Pull Request»** по участникам (с пометкой **«(Вы)»** у текущего пользователя).

### 5) Диапазоны времени жизни PR
- **Минимальное** время жизни Pull Request.
- **Максимальное** время жизни Pull Request.
- Кольцевая диаграмма **«Распределение времени жизни PR»** (группы по диапазонам, например: `< 1 часа`, `1–6 часов`, `6–24 часа`, …).
- Выделение доминирующего диапазона (например, **«< 1 часа»** — самое активное).

### 6) Список Pull Requests
- Заголовок блока: **«Список Pull Requests»**
- Действие **«Смотреть все»** — открывает полный список.
- Карточки PR с основными данными:
  - название PR
  - статус (например, **closed**)
  - автор/создатель
  - назначенные участники
  - дата создания и дата закрытия
  - мини-статистика (иконки комментариев/коммитов и т.п.)
  - ссылка на PR

### 7) Подборки
- **Топ-5 самых быстрых PR** (с указанием времени закрытия).
- **Топ-5 самых медленных PR** (с указанием длительности).

### 8) Итоги
- **Место в команде**.
- Итоговая **оценка Pull Request**.
